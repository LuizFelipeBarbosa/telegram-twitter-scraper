B · Canvas rewrite with d3-zoom Replace SVG rendering with <canvas>. d3-zoom writes transforms into canvas transform matrix. Requires hand-rolled hit testing for click/hover (quadtree). Much better performance at high node counts but a larger rewrite and loses DOM accessibility features.

C · Add a React zoom/pan library Wrap the map in an existing React zoom-pan component (e.g. react-zoom-pan-pinch, @visx/zoom). Library handles wheel, pinch, pan, and exposes a transform you apply to your SVG content. Keep d3 for the force simulation and drag behavior.
✓ Battle-tested interaction · less code · consistent cross-platform gestures
✗ New dependency · may fight with d3-drag on bubbles · less control over filter/propagation

Recommendation: A. It's the smallest change, keeps the current architecture, and d3-zoom is the canonical tool for exactly this. B is overkill unless we hit the SVG node limit; C adds a dependency for something d3 already does natively.
