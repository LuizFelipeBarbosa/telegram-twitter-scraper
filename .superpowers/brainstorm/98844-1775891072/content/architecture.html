Architecture: canvas visuals + DOM button overlay

Both layers share a single CSS transform, so d3-zoom updates them together.

Layered rendering

OUTER CONTAINER

LandscapeMap.tsx (React)
Holds d3-zoom state. Positioned in the Landscape grid cell. Mounts canvas, hit layer, overlay controls, and tooltip.

ZOOM LAYER · single CSS transform driven by d3-zoom

1. <canvas> — visual rendering
Redraws on every zoom change + every force-simulation tick. Paints bubbles (fill=phase, stroke=kind, size=score), relations when hovered, labels above a zoom/size threshold. Hand-rolled using 2D context.
2. <div class="hit-layer"> — accessibility + interaction
One <button data-node-id="..."> per bubble, absolutely positioned at the bubble's untransformed xy. Transparent background, size matches bubble diameter. Each button has:
aria-label="{display_name}, {kind}, score {score}"
tabindex="0"
onClick / onKeyDown (Enter + Space)
onMouseEnter / onMouseLeave / onFocus / onBlur
onPointerDown (for drag start)
Both canvas and buttons are inside the same transform-group, so a single transform update moves both. React renders the buttons once per snapshot; d3-zoom only touches the parent's transform CSS property on subsequent interactions.

OVERLAYS · outside the zoom layer (do not scale)

3. Zoom controls — HTML buttons top-right
+ / − / reset buttons. Call zoom.scaleBy() / zoom.transform(identity) imperatively.
4. Scale indicator — mono text bottom-left
"× 1.00" subscribed to zoom state.
5. Keyboard hint — mono text bottom-right
Tiny "+ − 0 ← → ↑ ↓" legend, dismissible.
6. NodeTooltip — fixed position, pointer-following
Same as today. Activated by hit-layer button onMouseEnter.
7. Preview popover — pinned on shift-click
Persists until user dismisses or shift-clicks another bubble. Shows summary, metrics, "Open detail →" link.

Event flow

Hover Button's onMouseEnter → parent receives node_id → state updates → canvas redraws relations + tooltip opens.
Click Button's onClick. If shiftKey → open preview popover. Else → navigate to /node/:kind/:slug.
Drag Button's onPointerDown starts drag mode. Pointermove updates a local fx/fy. Canvas redraws the single node at the drag position each frame. Pointerup → animate back to original position over 180ms. Drag events set event.stopPropagation() so d3-zoom pan doesn't fire.
Zoom / pan d3-zoom on the outer container. Wheel / pinch / drag-on-background. Emits transform → parent updates CSS transform on the zoom-layer → both canvas and hit-layer move together. No React re-render, just a style update.

Why this works

Canvas gives you the perf ceiling you wanted. The transparent button layer preserves screen-reader enumeration, keyboard focus, and standard DOM click/hover handling — which also means Testing Library's getByRole("button", { name: /US election/ }) works exactly the same as it would for SVG. Tests actually get cleaner than the current SVG version (no more dispatchEvent hack for React 19 synthetic handlers).

Single transform on the shared parent means zoom/pan is one CSS property update per frame — effectively free, even with thousands of buttons in the DOM.
